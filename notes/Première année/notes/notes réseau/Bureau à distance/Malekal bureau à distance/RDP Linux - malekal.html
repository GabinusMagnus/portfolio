Lorsque l’on administre un réseau, pour assister un utilisateur ou administrer un poste client ou même un serveur, on peut prendre la main à distance.
Sur Windows, par défaut, celle se fait à l’aide de RDP (Remote Desktop Protocol) ou bureau à distance.
Il peut aussi arriver d’avoir un poste client en Linux (Ubuntu, Mint, Fedora, Debian, …).
Dans ce cas là, on a besoin d’un client RDP afin de contrôler le PC à distance.

Ce tutoriel vous explique comment se connecter au bureau à distance à Windows à partir de Linux.

Remmina

Remmina est un client de bureau à distance qui prend en charge de nombreux protocoles d’accès à distance tels que RDP, VNC, NX, XDMCP et SSH.
Il vise à être utile pour les administrateurs système et les voyageurs, qui ont besoin de travailler avec de nombreux postes de travail et/ou serveurs distants.
Remmina est inclus dans la distribution Ubuntu Linux et est son client de bureau à distance par défaut.

Vous créez une session qui s’ajoute dans une liste.
Ensuite, il ne reste plus qu’à double-cliquer sur celle-ci pour se connecter à distance sur un PC ou serveur.

Plus de détails dans ce tutoriel :

Remmina : logiciel de bureau à distance RDP, VNC, SSH pour Linux

FreeRDP

FreeRDP est une implémentation gratuite du protocole de bureau à distance (Remote Desktop Protocol) suivant les spécifications Microsoft Open.
Cette implémentation fournit à la fois les applications client et serveur ainsi qu’une bibliothèque, ce qui permet à d’autres applications d’utiliser le protocole RDP. Aujourd’hui, nous nous intéressons à l’application client FreeRDP.
Enfin FreeRDP est un fork de rdesktop.

Pour l’installer sur Ubuntu avec APT :

sudo apt install freerdp2-x11

La syntaxe est de la forme suivante :

xfreerdp /v:nom_ordinateur_ou_adresse_ip /d:domaine /u:nomutilisateur

Puis pour prendre la main à distance sur le PC 192.168.0.22 avec l’utilisateur mak :

xfreerdp /v:192.168.0.22 /u:mak

Puis la connexion au bureau à distance Windows à partir de Linux s’effectue.

Vinagre

Vinagre est le client de bureau à distance par défaut pour le bureau GNOME.
Comme Remmina, il prend en charge certains protocoles de connexion: SSH, RDP, SPICE et VNC.
Cependant, comme rdesktop, Vinagre n’est plus maintenu depuis un certain temps.

Ce logiciel de bureau à distance ne gère pas les sessions et ne propose pas de système d’onglets.

  • Cliquez sur Connecter
  • Puis dans Protocole choisissez RDP
  • Ensuite saisissez le nom de l’ordinateur ou adresse IP dans Hôte
  • En bas réglez la mise à l’échelle et le nom d’utilisateur et domaine
  • Puis saisissez le mot de passe de connexion au bureau à distance
  • La connexion au bureau à distance Windows à partir de Linux s’établit

rdesktop

rdesktop est un ancien client RDP pour Linux mais qui ne semble plus maintenu.
Il ne supporte pas Network Level Authentication (NLA) et donc vous risquez de rencontrer des problèmes pour vous connecter à des versions récentes de Windows.

Sinon le désactiver dans les paramètres avancés du bureau à distance :

  • Ouvrez les paramètres de Windows 10
  • Puis système
  • A gauche cliquez sur bureau à distance
  • Puis au milieu paramètres avancés
  • Désactiver Faire en sorte que les ordinateurs utilisent l’authentification au niveau du réseau pour se connecter

Ensuite pour se connecter en RDP Avec rdektop :

  • Ouvrez un terminal
  • Puis saisissez la commande rdesktop suivi l’adresse IP du PC distant
rdesktop 192.168.0.22
